# قائمة اختبار روابط الإداريين والموظفين

**الموقع:** https://rewards-63e43.web.app

افتح كل رابط في **نافذة خاصة (Incognito)** أو بعد مسح الـ cache للنتيجة الصحيحة.

---

## 1) رابط بدون صلاحية

**الرابط:** `https://rewards-63e43.web.app/`

**المتوقع:** رسالة **"غير مصرح بالدخول"** + "يرجى استخدام الرابط الصحيح للوصول إلى النظام" + "إذا كنت موظفاً، يرجى استخدام QR Code الخاص بك"

---

## 2) رابط الأدمن (كامل الصلاحية)

**الرابط:** `https://rewards-63e43.web.app/?admin=ayman5255`

**المتوقع:** تظهر **صندوق الرفع** أو **لوحة التحكم** (لو فيه بيانات). أزرار "إدارة الإداريين"، "طباعة الكل"، "شروط المكافآت"، إلخ تظهر.

---

## 3) رابط إداري (مثل المشرف / الحسابات)

**الخطوات:**
1. افتح الموقع كأدمن: `?admin=ayman5255`
2. ارفع ملف وادخل لوحة التحكم
3. اضغط **"إدارة الإداريين"** → انسخ رابط أي دور (مثلاً المشرف)
4. افتح الرابط المنسوخ في **نافذة خاصة** أو من جهاز/متصفح آخر

**المتوقع:**
- لو الرابط صحيح (نفس الجهاز أو تم رفع الـ tokens لـ Firebase): تفتح **لوحة التحكم** مع صلاحيات ذلك الدور (مثلاً المشرف يرى "الكل" فقط، الحسابات يرى التقارير والطباعة بدون تعديل).
- لو الرابط خاطئ أو الفترة مغلقة: **"غير مصرح بالدخول"** أو **"جاري التحقق من الرابط..."** ثم رسالة خطأ مثل "الفترة غير موجودة" / "الرابط غير صحيح".

---

## 4) رابط الموظف (كود)

**الرابط:** `https://rewards-63e43.web.app/?code=XXXX`

استبدل `XXXX` بكود موظف حقيقي من **"أكواد الموظفين"** (بعد رفع ملف وفترة نشطة).

**المتوقع:**
- لو الكود صحيح وفيه بيانات (محلياً أو من Firebase): يظهر **تقرير الموظف** أو **اختيار الفرع** لو الموظف في أكثر من فرع.
- لو الكود خاطئ أو مفيش بيانات: رسالة أو شاشة توضّح عدم وجود بيانات / كود غير صحيح.

---

## ملخص من الكود (للتأكد من المنطق)

| الرابط | المسار في الكود | النتيجة المتوقعة |
|--------|-----------------|-------------------|
| بدون params | `doRbacThenInit` → `doAppInit` → `loadDataFromStorage` → `!isAdminMode() && !isRbacFromUrl` | "غير مصرح بالدخول" |
| `?admin=ayman5255` | `isAdminMode() === true` → تحميل البيانات وعرض لوحة التحكم | لوحة التحكم / صندوق الرفع |
| `?role=...&token=...&period=...` | `doRbacThenInit` → تحقق من localStorage ثم Firebase → `doAppInit` أو "جاري التحقق..." أو "غير مصرح" | حسب صلاحية الرابط والفترة |
| `?code=XXXX` | `loadDataFromStorage` ترجع مباشرة؛ `window.load` → `checkMobileEmployeeCode()` بعد 1 ثانية | تقرير الموظف أو "جاري تحميل البيانات..." ثم تقرير أو رسالة خطأ |
