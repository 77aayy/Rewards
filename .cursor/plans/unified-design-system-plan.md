# خطة توحيد المواصفات (Design System) — المشروع كامل

## الهدف
توحيد مواصفات الواجهة (ألوان، أنصبة، ظلال، مكونات) في **كل** صفحات ونوافذ المشروع، مع مصدر واحد للمواصفات ومراعاة الـ Rules.

---

## ما تم تنفيذه

### 1. مصدر واحد للمواصفات
- **الملف:** `app/shared/DESIGN-SYSTEM.md`
- يضم: متغيرات الثيم (داكن/فاتح)، متغيرات الصفحات القائمة بذاتها، مكونات مشتركة (بطاقة، أزرار، حقول، جداول)، وقائمة الملفات التي تحتوي الثيم.
- أي تغيير في المواصفات يُسجَّل هنا أولاً ثم يُنعكس في الملفات أدناه.

### 2. متغيرات CSS الموحدة
- **app/src/index.css:** إضافة `--adora-page-bg`, `--adora-standalone-card-*` لصفحات الدخول والبوابة.
- **app/public/rewards/src/styles.css:** نفس المتغيرات الإضافية.
- **app/Rewards/src/styles.css:** نفس المتغيرات (مزامنة مع rewards).

### 3. توحيد صفحات الدخول والبوابة
- **AdminLoginForm.tsx:** ثيم فاتح (تدرج خلفية، بطاقة بتدرج رمادي–أزرق، تركواز للعناصر البارزة).
- **AdminGate.tsx:** نفس الثيم (خلفية، بطاقة، حقول بيضاء، زر تركواز) ليكون متسقاً مع صفحة الدخول.

### 4. الملفات التي تعتمد على الثيم (مرجع)
| الملف | الاستخدام |
|-------|-----------|
| app/src/index.css | React: body, .glass, ثيم داكن/فاتح |
| app/public/rewards/src/styles.css | صفحة المكافآت: جدول، فلاتر، تقارير، نوافذ منبثقة |
| app/Rewards/src/styles.css | نسخة مزامنة من المكافآت |
| app/shared/action-header-buttons.css | أزرار الترويسة (--adora-*) |
| app/src/AdminLoginForm.tsx | صفحة تسجيل الدخول (ثيم فاتح ثابت) |
| app/src/AdminGate.tsx | بوابة المفتاح (ثيم فاتح موحّد مع الدخول) |

---

## خطوات مقترحة لاحقاً (اختياري)

1. **نوافذ منبثقة (Modals) في صفحة المكافآت:** التأكد من استخدام `var(--adora-modal-bg)`, `var(--adora-border)` في كل الـ modals (من app.js أو قوالب HTML).
2. **صفحات HTML منفصلة:** إن وُجدت (مثل clear-session.html) و under نفس المشروع، تطبيق نفس المتغيرات أو نفس ثيم البطاقة/الأزرار. (ملاحظة: clear-session.html مُبقاة متعمدة بدون ثيم لأنها شاشة عبور سريعة للتوجيه بعد تسجيل الخروج.)
3. **مراجعة دورية:** عند إضافة صفحة أو نافذة جديدة، الرجوع إلى `app/shared/DESIGN-SYSTEM.md` واستخدام `var(--adora-*)` دون ألوان ثابتة.
