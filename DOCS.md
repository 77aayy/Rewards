## 2. البناء والنشر

### البناء (محلياً)

```bash
npm run build
```

يُنتج مجلد `dist/` (يُستخدم إذا غيّرت `firebase.json` إلى `"public": "dist"`).

### النشر على Firebase Hosting

المشروع مُعدّ حالياً للنشر من **الجذر** (`public: "."` في `firebase.json`).

**تحديث النسخة الأونلاين (بعد أي إصلاحات):**
```bash
npm run deploy
```
أو يدوياً:
```bash
firebase use rewards-63e43
firebase deploy --only hosting
```

**تحديث الأونلاين ثم تشغيل الاختبار الكامل (يكشف دوائر غير مغلقة):**
```bash
npm run deploy:test
```

- تسجيل الدخول إن طُلب: `firebase login`
- Hosting URL بعد النشر: https://rewards-63e43.web.app
- الاختبار الكامل يشتغل ضد الأونلاين افتراضياً (`E2E_BASE_URL` غير مضبوط = https://rewards-63e43.web.app)

### (اختياري) النشر من `dist/`

1. في `firebase.json` غيّر `"public": "."` إلى `"public": "dist"`.
2. نفّذ: `npm run build` ثم `firebase deploy --only hosting`.

## 3. إعداد Firebase

### Storage Rules

- القواعد في `storage.rules`.
- رفع القواعد: `firebase deploy --only storage` أو نسخ المحتوى يدوياً من Firebase Console → Storage → Rules.

### تهيئة التطبيق

- الإعداد في `src/firebase-config.js` ويُحمّل من `index.html` لتهيئة مبكرة.
- التطبيق يكتب/يقرأ من Firebase Storage (مثلاً `periods/live.json` و`admin_tokens/...`).

### سلوك الفترة الحية (Firebase-First Architecture)

- **المصدر الوحيد للحقيقة:** Firebase Storage (`live.json` و `periods/{periodId}.json`).
- **localStorage:** يُستخدم فقط كـ **cache** — يُحدَّث من Firebase ويُقرأ منه فقط لو Firebase فشل (offline).
- **عند فتح أي صفحة (أدمن، مشرف، HR):** جلب الفترة من Firebase أولاً، ثم عرض اللوحة.
- **عند رفع ملف إكسيل:** جلب الفترة الحالية من Firebase، دمج (تحديث `count` فقط من الإكسيل، باقي البيانات تبقى)، ثم كتابة على Firebase.
- **عند إدخال تقييمات/حضور:** كتابة على Firebase بعد **150ms** من آخر تعديل (debounce سريع).
- **استطلاع الأدمن:** كل **2 ثانية** لعرض تحديثات المشرف/HR فوراً.

  1. **تقييدات المفتاح** في [Google Cloud Console](https://console.cloud.google.com/apis/credentials): HTTP referrers (rewards-63e43.web.app، localhost)، وربط المفتاح بـ Firebase Storage API فقط.
  2. **قواعد Storage** في `storage.rules`.

### مفتاح الأدمن (ADMIN_SECRET_KEY)

- **الموقع:** `src/app.js` — الثابت `ADMIN_SECRET_KEY = 'ayman5255'`.
- **تحذير:** المفتاح ظاهر في مصدر الصفحة؛ قبل نشر حقيقي غيّره إلى قيمة سرية ولا ترفعها لمستودع عام.
- الدخول كأدمن: `?admin=المفتاح`.

### تقييد API Key خطوة بخطوة (اختياري)

1. افتح [Google Cloud Console](https://console.cloud.google.com/) واختر مشروع rewards-63e43.
2. APIs & Services → Credentials.
3. عدّل المفتاح: Application restrictions = HTTP referrers، أضف `https://rewards-63e43.web.app/*` و`http://localhost:*`.
4. API restrictions = Restrict key → Firebase Storage API فقط.

## 5. اختبار E2E (Playwright)

### تثبيت وتشغيل

إذا ظهر `Cannot find module '@playwright/test'`:

```bash
npm install
npx playwright install chromium   # اختياري لأول مرة
```

- **ضد الأونلاين (بدون متغير):**  
  `npm run test:e2e`

- **ضد محلي:**  
  `set E2E_BASE_URL=http://localhost:3999` ثم `npm run test:e2e`

- **على Windows (بدون tee):**  
  `npx playwright test e2e/e2e-full.spec.js --project=chromium --reporter=line`  
  أو لحفظ الناتج: `npx playwright test e2e/e2e-full.spec.js --project=chromium --reporter=line | Out-File e2e-run.txt`

الملف: `e2e/e2e-full.spec.js`.

### قائمة التحقق (E2E حقيقي)

الاختبارات تُجرى **بالتسلسل** وكل رابط إداري (المشرف، HR، المدير، الحسابات) يُفتح في **متصفح موظف جديد** (`browser.newContext()`) — بدون تخزين سابق، محاكاة جهاز موظف لأول مرة.

1. أدمن: رفع ملف + فتح إدارة الإداريين لتفعيل الروابط على Firebase.
2. رابط المشرف يفتح على متصفح موظف جديد — **التحقق من عدم ظهور «تعذر تحميل بيانات الفترة»** ثم إدخال تقييمات وإرسال.
3. رابط HR يفتح على متصفح موظف جديد — نفس التحقق ثم إدخال حضور وإرسال.
4. التحقق عند الأدمن (متصفح جديد) من ظهور التقييمات والحضور من Firebase.
5. روابط المدير العام والحسابات تفتح على متصفح موظف جديد (بدون خطأ تحميل).
6. خروج ثم دخول — بقاء البيانات.
7. خصم من الأدمن والتحقق من بقاء التقييمات.

### إغلاق دائرة روابط الإداريين

لكي يعمل كل رابط (المشرف، HR، الحسابات، المدير العام) ويُحمّل بيانات الفترة:

1. **الأدمن:** بعد رفع ملف الفترة، يفتح «إدارة الإداريين» مرة واحدة — هذا يرفع التوكنات إلى Firebase (`admin_tokens/2026_01.json`) ويُحدّث الفترة الحية (`periods/live.json` و`periods/2026_01.json`).
2. **الإداري:** يفتح الرابط الذي نسخه الأدمن — التطبيق يتحقق من التوكن، ثم يجلب بيانات الفترة أولاً من `periods/2026_01.json` (الملف المُحدّث عند فتح «إدارة الإداريين»)، ثم من `live.json` إن لزم، ويعرض اللوحة مع الجدول.
3. إذا ظهرت «تعذّر تحميل بيانات الفترة» فتحقق من الاتصال بالإنترنت وأن الأدمن فتح «إدارة الإداريين» بعد رفع الملف.

### إكمال الاختبار اليدوي على الأونلاين (بعد رفع الملف والخصم)

إذا رفعت الملف وأضفت الخصم من «الخصومات» وظهر الخصم تحت «الخصومات المطبقة»:

1. **إدارة الإداريين:** من اللوحة اضغط «إدارة الإداريين» → تظهر نافذة بروابط المشرف و HR والحسابات والمدير العام.
2. **المشرف:** انسخ رابط «المشرف» من الحقل، افتحه في تاب جديد → اختر فرع (مثلاً الكورنيش) → أدخل تقييمات بوكينج/جوجل → «إرسال».
3. **HR:** انسخ رابط «HR» وافتحه في تاب جديد → اختر فرع → فعّل حضور (checkbox) → «إرسال».
4. **التحقق عند الأدمن:** ارجع لتاب الأدمن وحدّث الصفحة → تأكد أن الجدول يعرض التقييمات وأيام الحضور.
5. **إغلاق الفترة:** من اللوحة اضغط «إغلاق الفترة» → اقرأ التنبيه واضغط «تأكيد الإغلاق» → بعد الإغلاق تظهر صفحة الرفع من جديد.
